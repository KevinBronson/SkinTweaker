SETUP

This section will explain how to add and configure the SkinTweaker module to a page once it has been installed. Again, please refer to DotNetNuke's documentation for more detailed information on your particular version of DotNetNuke.

Instructions for DotNetNuke 6

  1. Create the page that you want SkinTweaker to be added to. Don't install any other modules on this page.
  2. Add the SkinTweaker Module to the page.
  3. You will find that 2 modules have been placed on the same page. One is named 'SkinTweaker Admin' and the other is named 'SkinTweaker Injector'.
  4. Open settings for the SkinTweaker Admin module.
  5. Go to Page Settings > Basic Settings. Select 'Host: DarkKnight - Invisible' for Module Container (You can try other containers if you wish, but this one is known to work).
  6. Click Update
  7. Open settings for the SkinTweaker Injector module.
  8. Again, go to Page Settings > Basic Settings. Select 'Host: DarkKnight - Invisible' for Module Container (You can try other containers if you wish, but this one is known to work).
  9. Go to Module Settings > Advanced Settings. Select the option 'Display Module On All Pages?'
  10. Click Update.
  11. Delete the module ('SkinTweaker Injector') that you just updated. This is done because you want to have the injector on every page EXCEPT for the SkinTweaker page, because the modules will interfere with each other.
  12. Go to Host > Settings > Advanced Settings > jQuery Settings. Check the option 'Use Hosted jQuery Version?'

Instructions for DotNetNuke 5?

 

Final Instructions for All Versions of DotNetNuke

  1. Go to the newly installed SkinTweaker page.
  2. On the left side of the screen you will see the 'Naviagtion Panel'. Find and click 'Advanced Settings'.
  3. Look for the section called 'Bug Hunter'.
  4. Click 'Kill All Bugs'.

That's it. Now you are ready to start using SkinTweaker.

 

 

UNINSTALL

If you wish to uninstall SkinTweaker, then you will likely want to restore any changes made by Bug Hunter. Once finished, you can uninstall the modules.

Restore Files Changed by Bug Hunter (Optional)

  1. Goto Navigation Panel's 'Advanced Settings' page.
  2. Find Bug Hunter.
  3. Click 'RESTORE SYSTEM'

Uninstall the SkinTweaker Modules

The standard uninstall process will remove the injector modules from all pages as well as the SkinTweaker module itself. Please refer to DotNetNuke's documentation for more detailed information on how to uninstall Extensions / Modules for your particular version of DotNetNuke.

OOPS! I forgot to use Bug Hunter's 'RESTORE SYSTEM' before I uninstalled.

No problem. Just reinstall SkinTweaker and then redo the uninstall process.
